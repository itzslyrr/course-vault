🩺 The Doctor

A read-only script that checks every note against your rules — catching what the guard can't.

What it is

The guard checks what the AI does. The doctor checks what the note says.

Your CLAUDE.md prose is the spec. The doctor turns each written rule into a check that can pass or fail. It only reads — it never edits a file — so it sails through the Day-6 guard hook. Run it with bash doctor.sh or just say "run the doctor."

How it runs

One loop over every note; four checks each; one verdict at the end.
1

Walk

Loop over every .md in knowledge/ and projects/*/ — one note at a time.

2

Read top 12 lines

Grab just the frontmatter block into a variable. All checks read from that.

3

Run 4 checks

Any failure prints a line and flips a flags counter from 0 → 1.

4

Verdict

flags still 0 → exit 0. Any problem → exit 2.

The flags trick: it starts at 0 ("healthy"). Every problem sets it to 1. At the end, 0 means clean, non-zero means "found something."

The four checks

Each rule in CLAUDE.md → one yes/no check, one report line naming file + fault.

1 · Frontmatter present

Every note has title, type, created, tags.

MISSING created: knowledge/broken-note.md

2 · Legal type

Type is one of reference · learning · decision · project · adr · work-item.

UNKNOWN type leraning: …

3 · Kebab-case name

Filename has no capitals, spaces, or underscores.

NOT kebab-case: My_Note.md

4 · No dead links

Every [[link]] points at a note that actually exists.

DANGLING missing-note: …

Live proof

We broke a note on purpose — misspelled type: leraning, no created, and a link to a nonexistent note:

# $ bash doctor.sh
MISSING created: knowledge/broken-note.md
UNKNOWN type leraning: knowledge/broken-note.md
DANGLING missing-note: knowledge/broken-note.md
# exit 2

# after fixing / removing it:
clean bill of health
# exit 0

Your tired eyes skim a typo. A check never does — same input, same answer, every time. That's deterministic.

The verdict, and why it matters

Exit codeMeaningEffect
exit 0clean bill of healthsafe to commit
exit 2found problemsfix first — the report is your repair list
This plugs the doctor into the daily routine → draft → doctor → diff → commit. By the time your eyes reach the diff, the mechanical errors are already gone — your review time goes to meaning.
