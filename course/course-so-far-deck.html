Building a Vault
the AI can't corrupt

Three weeks turning a pile of notes into a governed knowledge system — where the AI drafts fast, and every change is checked, and I sign what lands.

🎤 Say: "This is what I built over three weeks: a personal knowledge vault, and a set of rules that let an AI help me at speed without ever putting garbage — or unsigned changes — into it."
The Big Idea

Fast AI + human control, not one or the other

  • The AI drafts — notes, decisions, code — in seconds
  • Every write is checked before it lands (shape, rules, safety)
  • Nothing is real until I sign it (I review the diff, I commit)

The whole course is one question, answered five ways: where does enforcement live so it can't be bypassed?

🎤 Say: "The point isn't to slow the AI down — it's to let it move fast while a checkpoint I control catches anything wrong."

Week 3 · The Spine

The governed write path

your sentence → the skill → the counter (checked) → the doctor (form) → the vault (status: proposed) → your signature

Every step below is one lesson. Together they make this the only road a note can travel.

🎤 Say: "Everything in Week 3 builds this one path. Let me walk each piece — each is a day of the course."
Day 5 · The Promise

🤝 The house rule

Reads run free; writes wait for review. The AI's vault skill drafts a note, then STOPS — I read the diff and commit.

a promise the AI makes
🎤 Say: "Day 5: the AI promises to stop before a write. But a promise is just a line in its instructions — delete that line and nothing stops it."
Day 6 · The Law

🛡️ The guard hook

A PreToolUse hook that runs outside the AI, before every action. exit 2 = blocked, exit 0 = allowed. I proved it: a stolen git commit was refused with my own words.

a law the system enforces
🎤 Say: "Day 6 turns the promise into a law — a guard that checks the AI every time and can't be talked past."
Day 7 · The Inspector

🩺 The doctor

A read-only script that checks every note against the rules — missing frontmatter, unknown type, dead [[link]]. Same answer every time; tired eyes skim a typo, a check never does.

checks the files — after
🎤 Say: "Day 7: an automated inspector. It reads what a note SAYS — which the guard never does — and reports every flaw."
Day 9 · One Door

🚪 The doorway

A vault CLI — capture · adr · plan · task · done — that checks each write at the entry, before it lands. Refuse → reason + exit 2; accept → receipt + proposed. The AI's skill was rewired to use it.

checks the write — before
🎤 Say: "Day 9: a single guarded door. The doctor checked after; the doorway checks before — and both the human and the AI use it."
Day 10 · The Wall

🧱 The second law

The guard learns a second law: a direct Write/Edit into a note zone is blocked — "notes go through the doorway." Tools stay editable, reads run free. The side door is walled.

the door is now the only way in
🎤 Say: "Day 9's counter was opt-in — you could still bypass it. Day 10 bricks that hole shut: the door is the only way in."
Day 11 · The Guard at the Counter

🩺 The doctor at the counter

The counter now inspects the form before filing: required title · type repair · zone default · non-sensitive only. Refuse + write nothing, or file as proposed. Repair fixes shape, never content.

every note arrives well-formed
🎤 Say: "Day 11: the doctor moves INTO the door. Now the counter won't even file garbage — and 'non-sensitive only' is enforced, not just written on a wall."
