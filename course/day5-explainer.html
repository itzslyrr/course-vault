Skills, ADRs & Hooks

The three ideas behind today's lab — explained with the actual files in your ~/vault.

1 Skill — a tool you teach the AI once

A saved recipe with a trigger phrase. You write it once; it fires on its own when your sentence matches.

A skill is just a SKILL.md file. The description at the top is the trigger — the AI watches your words, and when they match, it runs the steps. No command to type.

--- your capture-note skill ---
name: capture-note
description: Use when the user says "save this", "capture this",
  "note this down", or "add this to my vault". ...
---
1. Decide the type   2. Create knowledge/<title>.md + frontmatter
3. Write body + [[link]]   4. Add to index   5. Show it and STOP

Today you build my-vault — one skill holding three recipes (called verbs): capture, record-a-decision, and recall. It picks which verb by matching your sentence.

Verb = one recipe inside a skill. my-vault has 3 verbs = 3 jobs it can do.

2 ADR — a record of a decision & why

Architecture Decision Record. One decision per file: what you chose, what you rejected, and why.

Six months later nobody asks "why did we build it this way?" — the answer is written down. You already have two, numbered so they form a timeline:

projects/task-tracker/adr-001-stack-choice.md   ← "we chose this stack"
projects/task-tracker/adr-002-newest-first.md   ← "show newest tasks first"

In step 5 you say "we decided to soft-delete tasks" → the skill writes the next one, adr-003-soft-delete-tasks.md, with status: proposed, and links it into the project.

A knowledge note says "here's how CORS works." An ADR says "here's a choice we made, and why." Different type — that's the only difference.

3 Hook — an automatic gate around the AI

A script the system runs by itself at a set moment — it can allow or block what the AI is about to do.

You don't call a hook; the system does. Day 5's hook is a guard that fires before a write (it sleeps until Day 6). Read it in English:

# vault-write-guard.sh
Read | Grep | Glob            → allowed, no questions   (reads run free)
Write | Edit into ~/vault    → flag it for review      (writes wait)

That is the house rule turned into code: reads run free, writes wait for your review.

🔑 The key idea: promise vs. wall

A skill is a promise the AI makes. A hook is a wall the system builds. Today you rely on the promise; Day 6 you build the wall.
That's your homework question: "what stops it if the STOP line is missing?" → the hook.
