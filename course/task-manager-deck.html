Three Foundations

Building a Task Manager — and learning that the interface you see is the least interesting part. A decisions-first walk from a browser list to a real database.

🎤 Say: "I built a task manager three times. The screen barely changed — but underneath, it went from a list in the browser to a Kanban board on a real server with a database. That gap is what I want to show you."
The Journey

Three versions, one idea getting deeper

v1

A list

Create · list · complete · delete. One screen, saved in the browser. Proves the core loop.

v2 · Kanban

A board

To Do → In Progress → Done. Cards move across columns; work-in-progress is limited.

v2 · real stack

A server

SvelteKit + SQLite. The data leaves the browser and lives in a database the server owns.

🎤 Say: "Each version keeps everything before it and adds one real idea — not more buttons, but a stronger foundation."
The Method

Every choice is a written decision (ADR)

Not "I coded it this way" — but "here's what I chose, what I rejected, and why." 13 ADRs across the versions.

# the decision set grew with the app
ADR-001  v1 stack: self-contained web app
ADR-002  newest-first ordering
ADR-004  v1 scope: CRUD only  # deciding what to LEAVE OUT
ADR-006  v2 is a Kanban board
ADR-009  WIP limit of 3 on In Progress  # the heart of Kanban
ADR-011  v2 stack → SvelteKit + SQLite
ADR-013  the WIP rule is enforced on the server
🎤 Say: "The ADRs are half the work. Even choosing what to leave out — no accounts, no sync in v1 — is a recorded decision, not an accident."

Version 2 · Research

What is Kanban?

  • Visualize the work — every task is a card on a board
  • Limit work-in-progress — cap how many cards can be "in progress" at once
  • Manage flow — watch cards move left → right; finish before you start

Origin: Toyota's factory floor. The WIP limit is what makes it Kanban, not just a list with columns.

🎤 Say: "Kanban's real idea isn't columns — it's limiting how much you start, so things actually get finished."

The Key Insight

Same screen. Different machine underneath.

 v2 · localStoragev2 · SvelteKit + SQLite
How it runsa file you double-clicka URL served by a server
Where data livestrapped in one browsera database file on the server
Who enforces the WIP rulebrowser JS (bypassable)the server (can't bypass)
Second browser sees…a separate empty boardthe same board
🎤 Say: "They look identical on purpose — so you can see the UI didn't change. Everything that matters for sharing, backup, and sync did."
Proof · The HTML

The page is built, not stored

What I wrote — the shell (356 bytes)
<body>
  <div>%sveltekit.body%</div>
</body>
# no board. just an empty slot.
What the server sends (12,136 bytes)
<div class="card done">
  <div class="txt">Ship Task Manager v1</div>
  <button>◀</button> …
# the slot is gone — filled from SQLite

356 → 12,136 bytes   built fresh on every request.

🎤 Say: "This 356-byte file is all I wrote for the page. The 12KB my browser receives is built by the server every request — it reads tasks from SQLite and fills the slot. The HTML isn't a file anymore; it's a live result."
Proof · The Rule

The server refuses — with no browser involved

# raw commands — no WIP code in them at all
card 1 → In Progress : HTTP 200  (allowed)
card 2 → In Progress : HTTP 200  (allowed)
card 3 → In Progress : HTTP 200  (allowed)
card 4 → In Progress : HTTP 409  ← BLOCKED by the server

# read the database: In Progress held at 3, not 4

The rule lives in the database layer, so it holds against any client — a tab, a phone, a raw command.

🎤 Say: "I moved cards with raw commands — no browser. The server still enforced the limit: a 409 on the fourth. The rule lives with the data, not in the page."
The Deeper Lesson

Enforcement lives where it can't be bypassed

  • The house rule — a promise the AI makes (Day 5)
  • The guard hook — a law outside the AI (Day 6)
  • The doctor — checks the files from outside (Day 7)
  • The server WIP limit — the rule lives with the data (this project)

Same idea, four times: a real rule belongs outside the thing it's guarding.

🎤 Say: "This ties the whole week together. A promise can be forgotten; a rule in the right place can't be bypassed."
