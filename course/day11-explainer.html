🩺 The Doctor at the Counter

The doorway stops filing garbage — it inspects the form before it writes.

What changed

The counter was the only door — but it filed whatever it was handed.

Day 7's doctor checked notes after filing; Day 9's doorway was the only way in. Day 11 puts the doctor at the counter: vault capture now validates the form before it writes. On a hard fail it refuses and writes nothing; on pass it files as status: proposed — every note arrives well-formed, ready to sign.

The four checks

Each refusal names the fault and shows the fix. Repair fixes shape, never content.

1 · Required title

an empty title is refused.

a note needs a non-empty title.

2 · Valid type (repair)

note/task/idea; a wrong-case near-miss is lowercased, not rejected.

Task → task

3 · Allowed zone (default)

inbox/ knowledge/ tasks/; empty defaults to inbox/.

'' → inbox/

4 · Non-sensitive only

anything marked sensitive is refused — the vault's prime rule, now enforceable.

non-sensitive content only.

Repair vs reject

vault capture "Call the vendor" --type Task   # wrong-case, no zone
→ captured: inbox/call-the-vendor.md (type: task, status: proposed)
   # Task → task, empty zone → inbox/ — filed clean

The doctor repairs shape (lowercases a type, defaults a zone) but never invents content — it will not write a title you didn't type.
