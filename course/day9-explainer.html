🚪 The Doorway

A CLI that checks every write at the entry — before it lands in the vault.

What it is

One command, one door. Both the human and the AI use it.

The doctor (Day 7) checked notes after they were filed. The doorway — a vault CLI at ~/vault/bin/vault — checks each write before it lands. Every write is a fail-closed transaction:

# refuse → a clear reason to stderr, exit 2, nothing written
# accept → a receipt to stdout, exit 0, note filed with frontmatter + index link

The verbs

Write verbs check before they write; the read verb never writes.

capture

a new knowledge note (checks the form)

adr

the next-numbered decision record

plan

a scope built on one or more ADRs

task

a work item — belongs to a plan

done

flip a work item to done

help

lists everything · refuses the unknown

Proven with three bad parcels

# the counter reads the form before it files it
vault capture ""                 → empty words · exit 2
vault capture "x" --type banana  → unknown type · exit 2
vault capture "(a duplicate)"    → already exists · exit 2

Three refusals, three clear door signs, fail-closed at the entry (Day 6's word, second verse).
