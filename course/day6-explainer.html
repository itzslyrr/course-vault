🛡️ The Guard Hook

Turning yesterday's promise into a law — a guard that runs before the AI acts, every time.

A skill is a promise the AI makes. A hook is a wall the system builds.

What a hook is

A script the system runs before every action the AI takes (PreToolUse).
1

AI wants to act

It's about to run a Bash command (e.g. git commit).

2

Guard runs first

The system pauses and hands the command to vault-write-guard.sh.

3

Verdict by exit code

exit 2 = blocked (with a reason) · exit 0 = allowed.

4

System obeys

On exit 2 the command is cancelled — it never runs.

The verdict is a number. exit 0 means "pass," exit 2 means "block." That single number is the whole decision.

The two laws we wrote

Each law matches the command's text and blocks with a human-authored reason.

Law 1 · the signature

Command contains git commit or git push → block.

"committing is the human's signature."

Law 2 · the delete

Command contains rm …vault → block.

"deleting in the vault needs your hands."

# vault-write-guard.sh (the heart of it)
input="$(cat)"                          # the AI's action, as JSON on stdin
cmd=$(... extract .tool_input.command)  # what is it about to run?
case "$cmd" in
  *"git commit"*|*"git push"*) echo "…signature…" >&2; exit 2 ;;
  *"rm "*vault*)             echo "…your hands…" >&2; exit 2 ;;
esac
exit 0   # everything else runs free

Wiring it in

A guard that isn't registered is a scarecrow. It's switched on in ~/.claude/settings.json:

"PreToolUse": [ { "matcher": "Bash",
    "hooks": [{ "command": "~/.claude/hooks/vault-write-guard.sh" }] } ]

In English: before any Bash action, run my guard. matcher: "Bash" means it only watches shell commands — your file reads and edits flow through untouched.

Live proof

The AI was told "commit this for me" and genuinely tried:

# AI runs: git commit -m "signed by the AI"
PreToolUse:Bash hook error: committing is the human's signature. Show the draft and stop.
# the commit never happened — git log is still empty

It didn't choose to stop this time. It was stopped. A read (ls ~/vault) still ran free — the guard only blocks what matches a law.

Fail-closed: when a guard is unsure, it blocks by default. The bridge stays up; lowering it is a deliberate choice.
