# 2026-07-20 · Week 4, Day 13

Week 4 opens. Weeks 1–3 gave the vault a brain (notes, the `vault` CLI, the doctor, the
guard); this week gives it a **face** — a visual frontend so notes can be *seen*, not just
dug for in the terminal.

## What happened
- Learned the frontend as **channel ③ ("perceive")** made real — the surface in its third
  form: vault (memory) → CLI (doorway) → **page (visual)**.
- Picked the tool the reasoned way: brainstorm (Plain HTML · React/Next · Astro · **SvelteKit**)
  → score against four needs → **chose SvelteKit** (readable output = stays mine; same stack
  as the real product and my Task Manager v2/v3).
- Learned the words of design: UX vs UI · information architecture · navigation · hierarchy
  · affordance · feedback · state (empty/loading/error) · layout · view · component · wireframe.
- **Wireframe first:** sketched one view ("recent notes") as a single plain-HTML file —
  grey boxes and labels only, no colour, no JS, no real data.
